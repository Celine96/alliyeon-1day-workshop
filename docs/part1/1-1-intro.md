
# 1부: 클로드 코드 세팅

# 1.1 클로드 코드란?

---

### [개념] 클로드 코드 = 내 컴퓨터에서 직접 일하는 AI 비서

일반적인 AI 챗봇(ChatGPT 등)은 대화만 해요.
클로드 코드는 달라요. 내 컴퓨터의 파일을 직접 읽고, 만들고, 수정하고, 명령을 실행해요.

알아야 할 명령어는 딱 하나: `claude`

### [개념] ChatGPT와 뭐가 다른가요?

각 기능을 좀 더 풀어볼게요:

1. 파일 접근 (File Access): ChatGPT는 대화창 안에서만 동작하지만, 클로드 코드는 내 컴퓨터의 폴더와 파일을 직접 다룰 수 있어요.

2. 기억력 (Memory): ChatGPT는 대화를 닫으면 이전 맥락을 잊지만, 클로드 코드는 CLAUDE.md라는 설정 파일에 프로젝트 정보를 저장해서 다음 대화에서도 기억해요.<br>
*(CLAUDE.md라는 메모리 파일에 저장합니다. 2부 2.5에서 자세히 다룰 예정입니다.)*

3. 실행력 (Execution): ChatGPT는 이메일 전송, 알람 생성처럼 지시한 단일 작업을 실행하지만, 클로드 코드는 지시한 목표를 스스로 분해하고 기획부터 구현까지 자동화해요.<br>
예를 들어 "마케팅 카드뉴스를 만들어줘"라고 하면, 데이터 분석 → 디자인 → 이미지 생성까지 스스로 판단하며 워크플로우를 구축합니다.

4. 동시 작업 (Parallel): ChatGPT는 하나의 요청을 순서대로 처리하지만, 클로드 코드는 여러 작업을 동시에 처리할 수 있어요.

5. 에이전트 (Agent): ChatGPT는 하나의 AI가 모든 것을 처리하지만, 클로드 코드는 기획·실행·검수처럼 역할별 전문 에이전트를 두고 분업할 수 있어요.<br>
*(2부 2.1에서 에이전트 개념으로 자세히 다룰 예정입니다.)*

> 핵심 차이: ChatGPT도 실무에 활용할 수 있지만, 새 대화를 시작할 때마다 프로젝트 맥락, 선호사항, 용어 등을 매번 다시 설명해야 합니다. 이것을 재설명 문제(Re-Explanation Problem)라고 해요. 클로드 코드는 CLAUDE.md에 이 정보를 저장해두기 때문에, 매번 처음부터 설명할 필요 없이 바로 작업을 이어갈 수 있습니다.

### [개념] Claude 제품군 — 생각·구축·실행

같은 회사(Anthropic)의 Claude AI를 세 가지 다른 환경으로 제공해요. 같은 프롬프트("이 데이터 읽고 상위 5가지 통찰 요약")라도 제품에 따라 결과가 달라져요.

#### Claude.ai — 대화 중심 / 생각 도구
> - **위치** — 웹 브라우저
> - **하는 일** — 구조·문구·추론 도움 → 생각의 틀 제공
> - **잘 맞는 일** — 질문·탐색·초안 작성·분석

#### Claude Code — 기술 환경 / 실행 도구 (← 오늘 우리가 쓸 도구)
> - **위치** — 터미널 (내 컴퓨터)
> - **하는 일** — 파일·스크립트를 직접 다룸 → 파일 결과물
> - **잘 맞는 일** — 파일 처리·자동화·콘텐츠 생성·검수

#### Claude Co-work — 비즈니스 도구 / 실행 도구
> - **위치** — 데스크톱 + Slack·문서·이메일
> - **하는 일** — 보고서 찾고·열고·요약본 작성 → 업무 완료
> - **잘 맞는 일** — 파일 처리·작업 조정·반복 다중 작업

### [개념] AI가 일하는 방식의 시대 흐름

AI 도구는 시대별로 **달성하려는 목적**이 달라졌어요.

> **🗓️ 2023 — Prompt 시대**
> - 핵심: "프롬프트를 잘 쓰면 좋은 답이 나옴"
> - 도구: ChatGPT, Claude 채팅, Gemini
> - 우리가 한 일: **한 줄 잘 쓰기**

> **🗓️ 2025 — Context 시대**
> - 핵심: "관련 자료(문서·데이터·이미지)를 다 넘겨주면 더 좋은 답"
> - 도구: Cursor, Manus
> - 우리가 한 일: **자료 다 모아주기**

> **⭐ 2026 — Harness 시대** (← 오늘 우리가 할 일)
> - 핵심: "AI가 일할 작업 환경 자체를 잘 짜기"
> - 도구: Claude Code, Codex
> - 우리가 할 일: **운영 시스템 만들기** (폴더·커맨드·에이전트 정의)

> 💡 '하네스(Harness)'는 2부 2.3에서 자세히 다뤄요.

### [개념] 왜 우리는 오늘 클로드 코드를 사용할까요?

#### 🌐 AI Native 흐름 — 비개발자도 기획부터 구현까지
> - 비개발자도 터미널 허들만 한 번 넘기면, **기획-구현-검수 에이전트를 각각 만들어 loop로 완성도 있는 결과물**을 만들어내는 추세
> - 다양한 도구 연동과 **커스텀 명령·에이전트 제작**으로 클로드 코드에서 기획부터 구현까지 AI Native 업무 자동화·위임이 가능

#### ✅ 클로드 코드의 강점 — 파일·직접 처리·md 전달
> 오늘은 **클로드 코드**에 집중해요. 이유:
> - 콘텐츠·데이터·자료 작업이 **파일과 폴더 중심**
> - 회사 내부 자료를 외부 도구 안 거치고 **내 컴퓨터에서 직접** 처리
> - 한 번 만든 에이전트가 **다음 사람에게 그대로 전달 가능** (md 파일 단위로)

> 💡 오늘 만들 카드뉴스 자동화(콘텐츠 에이전트 **Aria**)도 이 패턴의 작은 예시예요. 본격 적용 시 Planner / Generator / Evaluator 3-에이전트로 확장 가능 (2부 2.3 참고).

### [개념] 마인드셋 전환

|  | 기존 방식 | AI-Native 방식 |
|:---:|:---|:---|
| 사고방식 | "프로그래밍 언어를 배우고, 라이브러리를 익혀서 직접 구현해야 해" / "GPT 프롬프트를 잘 써야 좋은 결과가 나와" | "목적을 명확하게 전달하고, 역할을 분담시키면 AI가 구현해줘" |
| 구현 방식 | 사람이 코드를 한 줄씩 작성 / 프롬프트를 반복 수정하며 결과물 추출 | AI 에이전트가 기획부터 구현까지 자동으로 수행 |
| 학습 내용 | Python, JavaScript 등 코딩 언어 학습 / GPT 프롬프트 작성법 공부 | 원하는 결과를 정확하게 설명하는 방법 / AI에게 역할을 나눠주는 방법 |

이 워크숍에서 배울 것은 코딩이 아니에요. 목적을 명확하게 정의하고, AI에게 역할을 분담시켜서 기획부터 구현까지 자동화하는 방법을 배워요.
